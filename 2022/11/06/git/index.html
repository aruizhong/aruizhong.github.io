git 工作流程

工作区（写代码）==git add ==>

暂存区（临时存储）== git commit ==>

本地仓库（历史版本）== git push==> 远程库

git常用命令

说明 命令
查看帮助 git 操作 -help 或者 git 操作 --help
设置用户签名 git config --global[local] user.name xxx
git config --global[local] user.email xxx
global 是全局签名，local是某个仓库签名
git config user.name （查看签名
初始化本地库 git init 生成.git 目录
查看本地库状态 git status
添加到暂存区 git add xxx
git add . （全部添加
取消追踪 git rm --cached xx
git reset （还原最初状态，即退回全部
提交到本地库 git commit -m "提交信息" filename
git commit -m "提交信息" （提交全部
git commit -am '提交信息' （文件已被追踪，可以 -am
丢弃工作区的修改 git restore xxx
查看版本信息 git reflog
git log （更详细的版本信息
版本切换 git reset --hard 版本号
分支操作
创建分支 git branch 分支名
查看分支 git branch -v
切换分支 git checkout 分支名
合并分支 git merge 分支名
重命名分支 git branch -M 重命名
远程仓库操作
克隆仓库到本地 git clone 远程地址 [可以修改文件名]
拉取远程仓库到本地 git pull [别名] [分支]
推送到远程仓库 git push [别名][分支]
连接远程仓库 git remote add 别名 远程地址
查看远程仓库 git remote -v
